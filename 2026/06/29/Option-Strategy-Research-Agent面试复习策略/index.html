候选人视角：期权策略研究 Agent 项目面试复习策略

复习目标

这份文档的目标不是让你背代码，而是帮助你把这个项目讲成一个“懂金融业务、懂量化研究、懂 Agent 产品落地”的作品。

面试官真正想确认的是：

  • 你是否理解这个项目解决了什么金融投研问题。
  • 你是否能解释期权策略、Greeks、波动率、回测、因子挖掘这些核心概念。
  • 你是否知道 Agent 在项目里不是噱头，而是“规划 + 工具调用 + 结果生成”的业务流程自动化。
  • 你是否能承认项目边界，并提出合理的下一步优化。

一、先准备一段 1 分钟项目介绍

推荐表达

这个项目是一个期权策略研究 Agent MVP。用户可以输入类似“分析 AAPL 期权，给我一个保守收益型策略”的自然语言需求，系统会识别标的、市场观点、投资目标和风险偏好，然后自动获取市场数据和期权链，计算历史波动率、Black-Scholes Greeks，构建期权策略，做简化模拟，最后生成一份 Markdown 研究报告。

它不是实盘交易系统，也不是自动投顾，而是一个投研流程自动化原型。我的重点是把期权研究流程拆成可复用工具，再用 Agent 思路把这些工具编排起来。

你要主动强调的点

  • 这是研究工具，不是直接下单工具。
  • 金融计算是确定性的，不靠 LLM 编造。
  • Agent 负责理解需求、选择流程、调用工具、组织报告。
  • 当前是 MVP，真实投研还需要更强的数据、回测、风控和因子模块。

避免这样说

  • “这个系统可以自动赚钱。”
  • “我接入 AI 后它就能推荐最优策略。”
  • “回测结果证明这个策略一定有效。”

二、准备一段 3 分钟项目架构讲解

推荐讲解顺序

  1. 用户输入自然语言需求。
  2. RuleBasedPlanner 解析需求，形成 AgentPlan。
  3. AgentToolkit 调用数据、定价、策略、模拟和报告工具。
  4. 数据层获取市场价格、历史行情和期权链。
  5. 定价层计算 Black-Scholes 和 Greeks。
  6. 策略层构建 Covered Call、Protective Put、Bull Call Spread、Bear Put Spread、Iron Condor。
  7. 回测层做简化策略模拟。
  8. 报告层生成研究报告，可选用 LLM 做润色。

核心概念：Agent

Agent 可以理解为一个能根据目标自动组织行动的系统。它通常包含几个部分：

  • 目标理解：理解用户想做什么。
  • 规划：把目标拆成步骤。
  • 工具调用：调用外部函数、数据库、搜索、模型或计算模块。
  • 状态管理：记录当前做到了哪一步。
  • 输出总结：把结果整理成用户能理解的形式。

在这个项目里，Agent 不是一个完全自主的复杂智能体，而是一个 rule-based Agent MVP。它的价值在于把投研流程自动串起来。

核心概念：Planner

Planner 是 Agent 的“大脑雏形”。它负责把自然语言输入变成结构化计划。

例如用户输入：

Planner 可能识别出：

  • 标的：AAPL。
  • 市场观点：neutral。
  • 目标：income。
  • 风险偏好：conservative。
  • 推荐策略：covered_call。

核心概念：Tool Calling

Tool Calling 指的是让 Agent 调用外部工具完成具体任务。

在金融场景中，Tool Calling 非常重要，因为很多事情不能让大模型凭空生成，例如：

  • 市场价格必须来自数据源。
  • Greeks 必须通过公式计算。
  • 回测必须基于历史数据。
  • 报告里的数字必须可追溯。

这个项目里的工具包括：

  • 获取市场数据。
  • 获取期权链。
  • 计算历史波动率。
  • 计算 Greeks。
  • 构建期权策略。
  • 运行简化模拟。
  • 生成研究报告。

三、必须掌握的期权基础概念

1. 期权是什么

期权是一种衍生品合约，给买方在未来某个时间以约定价格买入或卖出标的资产的权利，但不是义务。

  • Call Option：看涨期权，买方有权以行权价买入标的。
  • Put Option：看跌期权，买方有权以行权价卖出标的。
  • Strike Price：行权价。
  • Expiration：到期日。
  • Premium：权利金，也就是期权价格。

2. 看涨期权 Call

Call 的价值通常来自标的价格上涨。

如果你买入 Call，标的价格越高越有利。最大亏损通常是支付的权利金，理论最大收益很高。

如果你卖出 Call，你收取权利金，但如果标的大幅上涨，会面临较大风险。Covered Call 中卖出的 Call 有正股覆盖，所以风险比裸卖 Call 小。

3. 看跌期权 Put

Put 的价值通常来自标的价格下跌。

如果你买入 Put，标的价格越低越有利。它常被用作下跌保护。

如果你卖出 Put，你收取权利金，但标的大跌时可能承担很大亏损。

4. 实值、平值、虚值

  • ITM：In the Money，实值期权，有内在价值。
  • ATM：At the Money，平值期权，行权价接近当前价格。
  • OTM：Out of the Money，虚值期权，没有内在价值，主要是时间价值。

例子：AAPL 当前价格 100。

  • 行权价 90 的 Call 是实值 Call。
  • 行权价 100 的 Call 是平值 Call。
  • 行权价 110 的 Call 是虚值 Call。

5. 权利金由什么决定

期权权利金通常受这些因素影响：

  • 标的价格。
  • 行权价。
  • 到期时间。
  • 波动率。
  • 无风险利率。
  • 分红。
  • 市场供需和流动性。

对期权策略来说，波动率和到期时间尤其重要。

四、必须掌握的四类策略

1. Covered Call

Covered Call 是“持有正股 + 卖出看涨期权”。

适用场景：

  • 已经持有股票。
  • 预期股价震荡或温和上涨。
  • 希望通过卖 Call 收取权利金。

收益来源：

  • 卖出 Call 收到的权利金。
  • 正股上涨到行权价之前的收益。

主要风险：

  • 如果股票大跌，正股多头仍然亏损。
  • 如果股票大涨，卖出的 Call 会限制上方收益。

面试表达：

Covered Call 不是无风险策略。它牺牲了部分上行收益，换取权利金收入，适合温和上涨或震荡市场。

2. Protective Put

Protective Put 是“持有正股 + 买入看跌期权”。

适用场景：

  • 长期看好标的。
  • 短期担心大跌。
  • 愿意支付保险成本。

收益和风险：

  • 如果标的大跌，Put 可以提供保护。
  • 如果标的上涨，正股仍有上涨收益。
  • 但 Put 的权利金会增加成本，抬高盈亏平衡点。

面试表达：

Protective Put 本质上是给股票仓位买保险。它不是为了增强收益，而是为了控制尾部风险。

3. Bull Call Spread

Bull Call Spread 是“买入低行权价 Call + 卖出高行权价 Call”。

适用场景：

  • 温和看涨。
  • 有明确上方目标价。
  • 不想支付太多权利金。

收益和风险：

  • 最大亏损是净权利金支出。
  • 最大收益是两个行权价差减去净权利金。
  • 上涨收益被高行权价 Call 封顶。

面试表达：

它比直接买 Call 成本更低，但收益也被限制，适合温和看涨而不是强烈看涨。

4. Iron Condor

Iron Condor 通常由四条腿组成：

  • 买入更低行权价 Put。
  • 卖出较低行权价 Put。
  • 卖出较高行权价 Call。
  • 买入更高行权价 Call。

适用场景：

  • 预期标的区间震荡。
  • 隐含波动率较高。
  • 没有重大突破事件。

收益和风险：

  • 收益主要来自卖出期权获得的净权利金。
  • 如果价格保持在区间内，策略更有利。
  • 如果价格大幅突破区间，可能亏损。
  • 买入两端保护腿是为了限制极端亏损。

面试表达：

Iron Condor 是卖波动策略，不是稳定套利。它怕大幅单边行情、跳空和事件冲击。

五、必须掌握的 Greeks

Delta

Delta 衡量期权价格对标的价格变化的敏感度。

如果一个 Call 的 Delta 是 0.5，标的价格上涨 1 元，期权价格大约上涨 0.5 元。

理解方式：

  • Call 的 Delta 通常为正。
  • Put 的 Delta 通常为负。
  • 正股 Delta 约等于 1。

在策略中：

  • Covered Call 通常仍然是正 Delta，但小于单纯持有正股。
  • Protective Put 的 Put 会降低组合整体 Delta。

Gamma

Gamma 衡量 Delta 对标的价格变化的敏感度。

它说明 Delta 会不会快速变化。Gamma 高的期权，在标的价格接近平值时，风险变化更快。

理解方式：

  • 买期权通常是正 Gamma。
  • 卖期权通常是负 Gamma。
  • 负 Gamma 策略怕大幅波动。

在策略中：

Iron Condor 这类卖期权策略通常会有负 Gamma 风险，所以怕标的大幅突破。

Theta

Theta 衡量时间流逝对期权价格的影响。

买期权通常是负 Theta，因为时间价值会衰减。卖期权通常是正 Theta，因为时间流逝对卖方有利。

理解方式：

  • 买保险要付时间成本。
  • 卖保险可以赚时间价值，但承担尾部风险。

在策略中：

  • Covered Call 卖出 Call，通常有一定正 Theta。
  • Protective Put 买入 Put，通常有负 Theta。
  • Iron Condor 通常希望时间衰减有利于自己。

Vega

Vega 衡量期权价格对波动率变化的敏感度。

如果 Vega 高，隐含波动率上升会显著推高期权价格。

理解方式：

  • 买期权通常是正 Vega。
  • 卖期权通常是负 Vega。
  • 卖波动策略怕波动率上升。

在策略中：

Iron Condor 通常希望隐含波动率下降或维持，不希望波动率突然升高。

Rho

Rho 衡量期权价格对利率变化的敏感度。

在短期期权策略里，Rho 通常不是最重要的风险；在长期期权中会更明显。

面试中可以这样说：

这个项目计算了 Rho，但对短周期策略来说，Delta、Gamma、Theta、Vega 通常更值得关注。

六、必须掌握的波动率概念

历史波动率

历史波动率是根据过去价格计算出来的波动程度。

它回答的是：

过去这只股票波动大不大？

常见计算方法是：

  • 计算每日收益率。
  • 计算收益率标准差。
  • 按 252 个交易日年化。

隐含波动率

隐含波动率是从期权市场价格反推出来的波动率。

它回答的是：

市场当前认为未来波动会有多大？

期权交易中，隐含波动率往往比历史波动率更关键，因为期权价格直接反映市场对未来波动的预期。

IV Rank

IV Rank 衡量当前隐含波动率在过去一段时间内处于什么位置。

例子：

如果过去一年 IV 最低是 20%，最高是 60%，当前是 50%，那么当前 IV 处在偏高位置。

用途：

  • IV Rank 高时，卖期权策略可能更有吸引力。
  • IV Rank 低时，买期权策略成本可能更便宜。

注意：

IV Rank 不能单独决定交易，还要看事件风险、趋势、流动性和价格区间。

波动率微笑

波动率微笑指不同执行价的期权隐含波动率不一样。

现实市场中，虚值 Put 往往更贵，因为投资者愿意为下跌保护付更高价格。

这说明 Black-Scholes 的“单一恒定波动率”假设并不完全符合真实市场。

波动率期限结构

期限结构指不同到期日的期权隐含波动率不同。

例子：

  • 财报前的近月 IV 可能很高。
  • 远月 IV 可能相对平稳。

对策略的影响：

如果近月 IV 很高，卖近月期权可能权利金更厚，但也可能代表重大事件风险。

七、Black-Scholes 要怎么讲

Black-Scholes 是什么

Black-Scholes 是经典的欧式期权定价模型。它用标的价格、行权价、到期时间、无风险利率和波动率来估算期权理论价格。

在这个项目中，它主要用于：

  • 计算 Call 和 Put 的理论价格。
  • 计算 Greeks。
  • 支持策略风险解释。

它的主要假设

  • 市场可以连续交易。
  • 没有交易成本和税费。
  • 无风险利率恒定。
  • 波动率恒定。
  • 标的收益服从对数正态分布。
  • 期权是欧式期权。

它的局限

真实市场中存在：

  • 交易成本。
  • bid/ask spread。
  • 流动性限制。
  • 波动率微笑。
  • 波动率期限结构。
  • 跳空风险。
  • 提前行权。
  • 财报和宏观事件冲击。

面试表达：

Black-Scholes 是一个很好的基础模型，但不能把理论价等同于真实成交价。真实投研还要结合隐含波动率曲面、盘口流动性和事件风险。

八、回测和模拟要怎么讲

回测是什么

回测是用历史数据模拟策略过去会如何表现。

它不能证明未来一定有效，但可以帮助我们理解：

  • 策略在不同市场环境下表现如何。
  • 最大亏损和回撤可能有多大。
  • 收益是否稳定。
  • 策略是否对参数过于敏感。

这个项目的模拟是什么级别

这个项目当前是简化模拟，不是机构级实盘回测。

它主要做了：

  • 使用历史价格。
  • 固定持有周期。
  • 根据策略 payoff 估算结果。
  • 输出累计收益、最大回撤、胜率等指标。

当前简化点

你要主动承认这些限制：

  • 没有完整处理 bid/ask spread。
  • 没有充分考虑滑点。
  • 没有考虑手续费和税费。
  • 没有考虑保证金。
  • 没有考虑提前行权。
  • 没有精细模拟期权合约滚动。
  • 没有做真实成交可行性验证。
  • 没有做多市场阶段样本外验证。

常见指标解释

累计收益：

策略总共赚了多少。

最大回撤：

从历史高点到低点的最大跌幅，衡量投资者可能承受的痛苦。

胜率：

赚钱交易占比。胜率高不代表策略好，因为小赚大亏的策略也可能胜率高。

夏普比率：

单位风险下的超额收益。越高通常越好，但要注意收益分布是否非正态。

盈亏比：

平均盈利和平均亏损的比例。

尾部风险：

极端行情下的大亏风险。期权卖方策略尤其要关注尾部风险。

九、因子挖掘要怎么准备

因子是什么

因子是可以解释或预测资产未来表现的变量。

在股票中，常见因子包括：

  • 动量。
  • 估值。
  • 质量。
  • 成长。
  • 波动率。
  • 流动性。
  • 情绪。

在期权中，还可以加入：

  • 隐含波动率。
  • IV Rank。
  • 波动率偏度。
  • 期限结构。
  • Put/Call Ratio。
  • Open Interest。
  • 成交量异常。

因子挖掘是什么

因子挖掘是寻找、构造、验证因子是否真的有预测能力或策略价值的过程。

基本流程：

  1. 提出金融假设。
  2. 收集数据。
  3. 构造因子。
  4. 做因子有效性检验。
  5. 做分组回测。
  6. 做样本外验证。
  7. 评估交易成本和换手。
  8. 判断是否能落地成策略。

IC 是什么

IC 是 Information Coefficient，通常指因子值和未来收益之间的相关系数。

如果 IC 长期为正，说明因子越高，未来收益可能越高。

如果 IC 不稳定，说明因子可能不可靠。

Rank IC 是什么

Rank IC 是用排名计算的 IC。它不关心因子值的绝对大小，而关心排序能力。

在量化选股中，Rank IC 很常用，因为我们通常关心因子能否把股票按未来收益排序。

分组回测是什么

分组回测是把股票或合约按照因子值分成几组，例如 5 组或 10 组，然后观察高因子组和低因子组未来收益是否有明显差异。

如果高因子组长期跑赢低因子组，因子可能有效。

样本外验证是什么

样本外验证是用没有参与调参的数据来测试策略。

它的意义是防止过拟合。

如果一个策略只在你调参的数据上表现好，在新数据上表现差，那很可能只是样本内自嗨。

未来函数是什么

未来函数是指在回测时使用了当时不可能知道的未来信息。

例子：

用未来一周的最高价决定今天是否买入，就是未来函数。

它会让回测结果虚假变好，是量化研究中的严重错误。

十、如何把因子挖掘和本项目结合

当前项目已有的研究链路

当前项目已经有：

  • 自然语言需求解析。
  • 市场数据获取。
  • 历史波动率计算。
  • 期权策略构建。
  • Greeks 分析。
  • 简化模拟。
  • 报告生成。

下一步可以补的因子

可以优先补：

  • Momentum：判断标的趋势强弱。
  • Historical Volatility：判断过去波动。
  • Implied Volatility：判断期权市场预期。
  • IV Rank：判断当前 IV 是否处于高位。
  • Skew：判断 Put 和 Call 哪边更贵。
  • Term Structure：判断近月和远月 IV 差异。
  • Liquidity：过滤成交差、价差大的合约。
  • Event Factor：识别财报、FOMC、CPI 等事件。

策略映射例子

如果趋势强、IV 较低：

可以考虑买 Call 或 Bull Call Spread。

如果持有正股、预期震荡、Call 权利金较高：

可以考虑 Covered Call。

如果持有正股、担心下跌、Put 保护成本可接受：

可以考虑 Protective Put。

如果价格区间震荡、IV Rank 较高、事件风险低：

可以考虑 Iron Condor。

面试表达

当前项目还没有做完整因子挖掘，但它已经具备因子扩展入口。后续我会把波动率、趋势、流动性、事件和期权偏度作为策略选择的输入，让 Agent 不只是根据关键词选策略，而是根据可验证的量化信号做策略筛选。

十一、RAG、Prompt Engineering 和 LLM 要怎么讲

RAG 是什么

RAG 是 Retrieval-Augmented Generation，中文可以理解为“检索增强生成”。

它的做法是：

  • 先从外部知识库检索相关资料。
  • 再把检索结果交给大模型生成答案。

金融 Agent 中，RAG 可以用于：

  • 检索公司财报。
  • 检索研究报告。
  • 检索策略说明。
  • 检索风险规则。
  • 检索历史案例。

Prompt Engineering 是什么

Prompt Engineering 是设计提示词，让模型按照指定格式、约束和目标输出。

在金融项目里，Prompt 不应该只追求语言漂亮，还要要求：

  • 不编造数据。
  • 保留风险提示。
  • 数字必须来自工具结果。
  • 无法判断时要说明不确定。
  • 不给确定性投资建议。

LLM 在本项目中应该做什么

适合 LLM 做的事：

  • 理解用户问题。
  • 生成研究计划。
  • 选择需要调用的工具。
  • 解释策略逻辑。
  • 整理报告语言。
  • 回答用户追问。

不适合 LLM 直接做的事：

  • 编造行情数据。
  • 猜测期权价格。
  • 直接生成回测收益。
  • 替代 Greeks 公式。
  • 给出无风险交易建议。

面试表达

我的设计思路是让 LLM 做“语言理解和流程编排”，让确定性工具做“金融计算和数据处理”。这样既能发挥 AI 的交互能力，又能降低金融幻觉风险。

十二、项目不足和改进路线

当前不足

你要主动说出这些不足：

  • Planner 是规则型，语义理解还比较简单。
  • 策略选择主要基于关键词和风险偏好，还不是完整量化信号。
  • 期权合约筛选逻辑较粗糙。
  • 回测没有充分考虑真实交易成本和流动性。
  • 没有完整因子挖掘模块。
  • LLM 目前只做可选报告润色，没有形成完整对话式 Agent。

改进路线

第一阶段：提高金融可信度。

  • 接入更可靠的期权链数据。
  • 清洗 bid/ask、volume、open interest。
  • 加入 IV、IV Rank、skew、term structure。
  • 完善交易成本、滑点和流动性过滤。

第二阶段：提高量化研究能力。

  • 加入因子库。
  • 做分组回测、IC、Rank IC。
  • 做 walk-forward 和样本外验证。
  • 做多策略横向比较。

第三阶段：提高 Agent 产品能力。

  • 引入 LLM planner。
  • 增加 RAG 财报和新闻检索。
  • 增加 Risk Agent。
  • 增加 Report Agent。
  • 增加审计日志和可追溯报告。

十三、常见面试问题与候选人答案模板

问题 1：这个项目和普通量化 notebook 有什么区别？

答案模板：

普通 notebook 通常是一次性分析，而这个项目把投研流程产品化和工具化了。它有自然语言入口，有 planner 做意图解析，有工具层调用数据、定价、策略、回测和报告模块，最后输出结构化报告。所以它更像一个投研 Agent MVP，而不只是单个研究脚本。

问题 2：为什么不用 LLM 直接推荐策略？

答案模板：

金融场景里，LLM 直接推荐策略容易产生幻觉，也很难审计。我更倾向于让 LLM 做需求理解和报告解释，把价格、Greeks、回测和风险这些关键环节交给确定性工具。这样输出更可复现，也更符合投研场景的风控要求。

问题 3：你如何证明这个策略不是拍脑袋？

答案模板：

当前项目是 MVP，策略选择规则还比较简单，但它已经把策略选择逻辑显式化了，例如保守收益型对应 Covered Call，下跌保护对应 Protective Put，温和看涨对应 Bull Call Spread。后续我会加入 IV Rank、趋势、偏度、流动性和事件因子，用量化信号来替代单纯关键词规则，并通过回测验证策略有效性。

问题 4：这个项目最大的风险是什么？

答案模板：

最大的风险是用户误以为它可以直接用于实盘。实际上当前项目是研究原型，回测和数据都做了很多简化，没有完整考虑滑点、手续费、bid/ask、保证金、提前行权和流动性。因此我在报告里保留了风险提示，并把它定位为教育和研究工具。

问题 5：如果让你继续做两周，你会优先做什么？

答案模板：

我会优先做三件事。第一，增强真实期权链清洗和合约筛选，包括 bid/ask、volume、open interest。第二，加入 IV Rank、skew、term structure 和 momentum 因子，让策略选择更量化。第三，增强回测，加入交易成本、滑点和参数敏感性分析。这样最能提升项目的金融可信度。

十四、面试前 3 天复习计划

第 1 天：讲清项目

要完成：

  • 背熟 1 分钟项目介绍。
  • 背熟 3 分钟架构介绍。
  • 能画出项目流程：输入、Planner、Toolkit、数据、定价、策略、回测、报告。
  • 能说清楚 Agent 体现在哪里。

自测问题：

  • 如果面试官完全不懂代码，我怎么解释这个项目？
  • 如果面试官问“这和 ChatGPT 有什么区别”，我怎么回答？

第 2 天：补金融和量化概念

要完成：

  • 理解 Call、Put、行权价、到期日、权利金。
  • 理解 Covered Call、Protective Put、Bull Call Spread、Iron Condor。
  • 理解 Delta、Gamma、Theta、Vega、Rho。
  • 理解历史波动率、隐含波动率、IV Rank、skew、term structure。
  • 理解回测、最大回撤、夏普、胜率、尾部风险。

自测问题：

  • 为什么 Covered Call 不是无风险？
  • 为什么 Iron Condor 怕大行情？
  • 为什么卖期权要关注波动率和尾部风险？

第 3 天：准备追问和扩展

要完成：

  • 准备项目不足和改进路线。
  • 准备因子挖掘扩展方案。
  • 准备如何引入 LLM、RAG、多 Agent。
  • 准备真实落地需要补哪些数据和风控。

自测问题：

  • 如果面试官问“你做过因子挖掘吗”，我如何把这个项目延展过去？
  • 如果面试官质疑“这个 Agent 太简单”，我如何回答？
  • 如果面试官质疑“这个回测不能用”，我如何承认并说明下一步？

十五、候选人最终表达策略

你要呈现的人设

你要呈现的是：

  • 懂金融业务逻辑。
  • 懂期权策略风险收益。
  • 懂量化研究基本流程。
  • 懂 Agent 产品怎么落地。
  • 知道 MVP 和真实生产系统的差距。

你不要呈现的人设

不要呈现成：

  • 只会调 API 的人。
  • 只会写 UI 的人。
  • 把 demo 当实盘系统的人。
  • 不懂风险却推荐策略的人。
  • 只会说大模型但不懂金融计算的人。

最重要的一句话

这个项目的核心价值不是“用了 AI”，而是把期权投研流程拆成可验证的工具，再用 Agent 思路自动编排起来；LLM 可以增强交互和报告，但金融计算、风险评估和回测必须保持可复现、可审计。
